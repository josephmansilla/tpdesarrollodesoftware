main -> proyecto base inicial

entrega-1 -> todos los commits para llegar al los entregables

entrega-N -> hasta la cantidad de entregas que haya

Para unos features específicos se puede crear una branch que salga directamente de la branch de la entrega actual 
sobre la que se está trabajando. Esta rama para este feature luego se mergea con entrega1 para mantener consistente el trabajo.

Al terminar una entrega se mergea en main los cambios finales de esa entrega con sus correspondientes correciones. 
Para empezar la siguiente entrega se puede crear una nueva rama con el nombre de la entrega a trabajar, desde la entrega n - 1 o main.
